## HTTP Request Standard Header

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| trace | string | Yes | Unique trace ID generated by requester (max 64 chars) |
| data | object | Yes | Data body (format defined per endpoint) |

Request example:

```json
{
  "trace": "c2a8a146-a647-4d6f-ac07-8c4805bf0b74",
  "data": {}
}
```

## HTTP Response Standard Header

| Field | Type | Description |
|-------|------|-------------|
| ret | int32 | Error code |
| msg | string | Success/failure description |
| trace | string | Same as request trace |
| data | object | Response data body |

Response example:

```json
{
  "ret": 200,
  "msg": "ok",
  "trace": "c2a8a146-a647-4d6f-ac07-8c4805bf0b74",
  "data": {}
}
```

## WebSocket Request Format

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| cmd_id | integer | Yes | Protocol command ID |
| seq_id | integer | Yes | Request sequence ID (echoed in response) |
| trace | string | Yes | Trace ID for logging |
| data | object | Yes | Data body |

## Error Codes

| Code | Description |
|------|-------------|
| 200 | Success |
| 202 | Invalid request parameters |
